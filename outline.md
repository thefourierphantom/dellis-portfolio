## Page Breakdown

### 1. index.html - Landing Page
**Purpose**: Professional introduction and site navigation
**Sections**:
- Hero area with investigator name and title
- Mission statement focused on digital evidence and investigations
- Navigation to key sections (Case Files, About, Resume, Contact)
- Professional background summary
- Recent case highlights (sanitized)

### 2. about.html - About the Investigator
**Purpose**: Professional background and investigative focus
**Sections**:
- Cybersecurity student with electrical engineering background
- Specialization in cybercrime investigation and digital forensics
- Evidence analysis and event reconstruction expertise
- CompTIA Security+ certification status
- Investigative interests and career trajectory
- Professional philosophy and approach

### 3. case-files.html - Case File Index
**Purpose**: Organized investigation portfolio with filtering
**Sections**:
- Case file filtering system (by category, status, evidence type)
- Grid of case cards with Case ID, scenario summary, evidence examined
- Categories: Network, Endpoint, Memory, OSINT investigations
- Search functionality for case discovery
- Each case card links to detailed investigation page

### 4. Individual Case Pages (case-001.html through case-004.html)
**Purpose**: Detailed forensic investigation reports
**Template Structure**:
- Case ID and classification
- Scenario Summary (2-4 sentences, sanitized)
- Evidence Examined (PCAPs, logs, artifacts - generic descriptions)
- Investigative Questions
- Methodology (high-level, non-operational)
- Findings (bullet points, evidence-based)
- Timeline Summary
- Conclusion
- Limitations & Confidence Level
- PDF Download option (redacted report)

### 5. skills.html - Skills & Methods
**Purpose**: Professional capabilities demonstration
**Sections**:
- Digital Evidence Analysis competency
- Network Traffic Analysis proficiency
- Endpoint Artifact Interpretation skills
- Incident Reconstruction capabilities
- Investigative Reporting expertise
- Interactive skills matrix with experience levels

### 6. credentials.html - Credentials & Education
**Purpose**: Academic and professional qualifications
**Sections**:
- CompTIA Security+ certification
- Cybersecurity BS degree (expected graduation)
- Relevant coursework and laboratory experience
- Professional development activities
- Academic achievements and recognition

### 7. resume.html - Professional Resume
**Purpose**: Comprehensive professional summary
**Sections**:
- Embedded PDF resume viewer
- Professional summary
- Education and certifications
- Technical skills and competencies
- Relevant experience and projects
- Professional references available

### 8. contact.html - Professional Contact
**Purpose**: Professional communication channel
**Sections**:
- Professional email address
- LinkedIn profile link
- Contact form with inquiry types
- Professional consultation availability
- Response time expectations
- Location and time zone information

## Interactive Components

### Case File Filtering System
- Filter by investigation type (Network, Endpoint, Memory, OSINT)
- Filter by completion status (Active, Completed, Under Review)
- Filter by evidence type (PCAP, Logs, Artifacts, Memory)
- Search functionality for case content

### Evidence Timeline Viewer
- Interactive timeline on case pages
- Clickable events revealing detailed analysis
- Visual markers for different evidence types
- Investigation phase progression indicators

### Skills Assessment Matrix
- Interactive competency grid
- Hover details for specific capabilities
- Professional development timeline
- Methodology expansion on click

### Contact Form with Validation
- Professional inquiry categorization
- Form validation with clear error handling
- File upload for case consultation (optional)
- Confirmation and response expectations
